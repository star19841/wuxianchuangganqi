# 模型引擎与接口管理设计说明

## 目标

在当前后台管理系统基础上，继续完成 `codingPrompt.md` 中的任务三，新增：

- 模型引擎管理
- 接口管理

两者都作为后台一级功能页面接入现有 Layui 管理壳。

## 页面结构

左侧导航新增两个一级菜单：

- 模型引擎
- 接口管理

页面进入方式：

- `模型引擎` 进入模型配置与测试页面
- `接口管理` 进入第三方接口配置与联通性测试页面

## 一、模型引擎管理

### 功能要求

- 支持新增 / 删除 / 修改 / 查询 / 分页（6 条 / 页）
- 列表采用橱窗卡片模式，1 行 2 列
- 支持设置默认模型
- 支持调整模型参数
- 支持测试对话

### 模型字段

模型配置表至少包含：

- `id`
- `name`
  - 页面中的模型配置名称
- `api_key`
- `api_url`
- `model_name`
- `temperature`
- `max_tokens`
- `is_default`
- `created_at`

### 默认模型

- 系统启动后应确保存在一条默认模型配置
- 默认写入以下配置：
  - `api_key = sk-aigc-1a1e29be656757152aee327e2c3481355c3b5596`
  - `api_url = https://aigc-api.aitoolcore.com/api/v1`
  - `model_name = qwen3.5-flash`
- 任意时刻只允许一条配置为默认模型

### 模型测试

- 提供输入框用于输入测试消息
- 提供按钮执行测试对话
- 默认优先使用当前默认模型
- 允许对指定模型发起测试
- 页面中展示测试结果或失败原因

### 运行边界

- 模型调用通过 OpenAI 兼容接口完成
- 如果运行环境无法联网，不影响模型配置管理本身
- 测试对话失败时要明确提示失败原因

## 二、接口管理

### 功能要求

- 支持新增 / 删除 / 修改 / 查询 / 分页（20 条 / 页）
- 支持第三方接口配置管理
- 支持接口测试与联通性检查

### 接口字段

接口配置表至少包含：

- `id`
- `name`
- `category`
  - 例如：天气 / 音乐 / 新闻
- `base_url`
- `method`
- `headers_json`
- `sample_params`
- `description`
- `is_enabled`
- `created_at`

### 测试能力

- 支持从后台对单条接口发起联通性检查
- 支持显示请求结果摘要或错误信息
- 第一版重点是配置管理和联通性测试，不强行内建复杂业务消费逻辑

## 三、架构落点

### Model 层

新增或扩展以下数据能力：

- 模型配置表
- 接口配置表
- 模型仓储
- 接口仓储

### Controller 层

新增：

- 模型引擎控制器
- 接口管理控制器

职责包括：

- 页面渲染
- 新增 / 编辑 / 删除提交
- 搜索与分页参数解析
- 模型测试调用
- 接口联通性测试调用

### View 层

新增页面：

- 模型引擎页
- 接口管理页

界面要求：

- 与当前 Layui 后台壳统一
- 模型页采用 2 列卡片模式
- 接口页采用更偏表格 / 配置列表型工作区
- 所有提示、确认、操作反馈尽量统一为 Layui 风格

## 四、测试要求

至少验证：

- 默认模型自动写入
- 模型配置新增与默认切换
- 模型列表分页与搜索
- 接口配置新增 / 编辑 / 删除 / 分页
- 页面可正常渲染
- 如果网络可用，模型测试与接口联通性测试可执行
- 如果网络不可用，失败信息可被捕获并反馈

## 五、范围边界

本次不包括：

- 长会话上下文管理
- 多轮模型聊天历史存储
- 接口调用数据缓存
- 复杂工作流编排
- 用户级模型权限控制

本次聚焦：

- 管理闭环
- 测试入口
- 默认配置能力
