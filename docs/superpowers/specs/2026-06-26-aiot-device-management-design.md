# AIOT 设备管理设计说明

## 目标

在现有 Tornado MVC 后台系统基础上，继续完成 `codingPrompt.md` 中的设备管理任务，并将后台界面调整为带左侧导航的管理系统。

本次范围包括：

- 左侧导航包含 `用户列表` 与 `设备管理`
- `用户列表` 保留现有能力并作为后台一级页面
- 新增 `设备管理` 一级页面
- 设备管理支持新增、删除、修改、查询、分页
- 设备列表按 `1 行 2 列` 的卡片橱窗模式展示
- 设备列表分页为 `6 条 / 页`
- 设备支持动态维护多条传感器配置

## 交互结构

### 后台导航

- 左侧导航固定为后台入口区域
- 菜单项至少包含：
  - 用户列表
  - 设备管理
- 点击不同菜单进入不同页面，而不是只在单页里临时切块

### 设备管理页

- 顶部提供搜索入口
- 主内容区展示设备卡片列表
- 右侧或下方提供新增 / 编辑设备表单
- 表单中可动态新增多条传感器配置行

## 数据模型

### 设备主表 `devices`

- `id`
- `box_id`
  - 设备唯一编码
  - 唯一约束
- `esp32_ip`
- `manage_url`
  - 管理地址，例如 `http://esp32wifiip:port`
- `device_name`
- `category`
- `created_at`

### 设备传感器子表 `device_sensors`

- `id`
- `device_id`
  - 关联 `devices.id`
- `sensor_name`
- `pin_code`
- `pin_remark`

说明：

- 一个设备对应多条传感器记录
- 传感器配置采用结构化子表存储，不使用文本拼接，也不使用 JSON 整体存储

## 功能约束

### 新增设备

- 录入设备基础信息
- 动态新增多条传感器配置
- 保存时主表与子表一并写入
- `box_id` 必须唯一

### 编辑设备

- 可修改设备基础信息
- 可修改已有传感器配置
- 可删除已有传感器配置
- 可继续新增新的传感器配置

### 删除设备

- 删除设备时同步删除其传感器配置

### 搜索与分页

- 支持按 `box_id`、设备名称、类别、ESP32 IP 等信息查询
- 每页展示 `6` 条设备
- 展示样式为 `2 列卡片`

## 架构落点

### Model 层

- 在 `app/models/db.py` 中补充设备表与传感器子表初始化
- 新增设备仓储模块，负责：
  - 设备新增
  - 设备编辑
  - 设备删除
  - 列表分页
  - 搜索
  - 设备详情读取
  - 传感器子项维护

### Controller 层

- 新增设备管理控制器
- 负责：
  - 设备列表页面渲染
  - 新增提交
  - 编辑提交
  - 删除提交
  - 搜索与分页参数解析

### View 层

- 保留现有后台布局风格
- 补齐左侧导航
- 设备管理页采用 Layui 风格卡片布局
- 表单区支持动态传感器行

## 测试要求

至少验证以下行为：

- 创建设备成功
- `box_id` 唯一约束生效
- 设备与传感器配置可正确读取
- 删除设备时传感器配置同步删除
- 设备列表分页与搜索生效

## 范围边界

本次不包括：

- 设备实时在线状态检测
- 设备网络连通性探测
- 设备远程控制能力
- 传感器图表、监控看板或历史数据分析

本次只完成后台设备资料管理能力。
